IV. Experiments

Textual caption conditioning

We explored whether adding a textual caption conditioning could improve the flow matching geolocation model. The original Riemannian Flow Matching architecture conditions only on StreetCLIP image embeddings [8]. We extended this setup by adding a second conditioning branch based on automatically generated image captions.

As shown in the architecture diagram, the image and text embeddings are concatenated before being projected into the flow matching conditioning space. We experimented with several captioning strategies using Qwen3-VL and Gemma4 models. The captions ranged from short free-form geolocation cues to more structured descriptions containing attributes such as terrain, vegetation, road markings, visible script, and infrastructure.

We also tested multiple ablations:

  • image + text conditioning
  • text-only conditioning
  • single-field structured captions
  • targeted VQA-style captions generated from explicit geolocation-related questions

We observed that caption quality and prompting strategy clearly affect performance. However, none of the caption-conditioned variants outperform the image-only baseline. We believe this is because the captions are generated from the same input image already used by the visual encoder, and therefore bring limited additional information. As a result, we did not pursue caption conditioning further.
